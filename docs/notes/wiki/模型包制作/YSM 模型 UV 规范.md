## YSM 模型 UV 规范

### 背景

Blockbench 中的 UV 处理在某些情况下很反直觉，而且很难在实际的游戏渲染中高效的复现，自 ysm 2.2.2 版本以来已一年之久，每个版本在这方面都有大大小小的问题。此次 2.5.4 snapshot2 版本再次尝试修复了一些 uv 处理方面问题，并发布此文章尝试纠正建模过程中容易导致渲染错误的行为。

### 适用情况

- 此规范仅适用于 YSM 2.5.4 或更高版本；

- 此次改动仅影响 2.2.1 之前版本导出的加密模型、2.5.4 之后的加密模型和所有未加密的模型，其他版本的加密模型仍然保持旧的处理方式；

- 此规范在 2.5.4 版本正式发布之前可能会不断修改，期间不会保证向后兼容性；一旦 2.5.4 版本正式发布，后续若再有修改会另起文章，并参照上一条尽力对旧版本的加密模型保持旧的处理方式；

- 如果你的模型严格遵守以下规范，但游戏内渲染效果仍然和 blockbench 不一致，那么这种情况一律视为模组 bug，可直接向开发组反馈；

- 如果你的模型明确违反以下规范，但游戏内的渲染效果却符合预期（不论是你自己的预期还是 blockbench 的预期），那么仍然建议你修正违反规范的部分。因为 ysm 后续的更新不会维护这种“侥幸正确”的情况，你的模型可能会在后续的新版本中出现渲染错误，并且 ysm 不会修复这种错误。

### UV 最小尺寸

==对于面积不为 0 的面，其 UV 尺寸的绝对值不得小于一个 UV 单位，更不能为 0==

这一条分为箱型 UV 和逐面 UV 两种情况：

#### 箱型 UV

箱型 UV 的尺寸无法手动设定，而是通过块尺寸计算。但在一些情况下其 UV 尺寸被计算为 0。所以：

==对于使用箱型 UV 的块，其尺寸不得小于 1，也不得小于 0，但可以等于 0==

注意：

- 如果为了避免其他因素造成的渲染错误而把尺寸设为诸如 0.0001 之类接近 0 的正数值，也是符合规范的；但不推荐这么做，因为面积趋于 0 的面仍然会参与渲染，进而产生不必要的性能开销，建议改用逐面 UV 手动移除不需要渲染的面；

  ![pZaN9wq.webp](https://s41.ax1x.com/2026/01/04/pZaN9wq.webp)

- 如果需要小于 1 的块（包括小于 0 的负尺寸块）来实现预期效果，可以使用大于 1 的块+小于 0 的膨胀值来实现，或者改用逐面 UV 手动设定符合规范的 UV 尺寸即可使用任意块尺寸。


#### 逐面 UV

对于使用逐面 UV 的块，只需确保每个面都满足上述要求即可。注意如果手动把尺寸小于 1 的箱型 UV 块转换为逐面 UV，其每个面的 UV 大小会被自动设定为 0。这是违反规范的，需要手动修正。
